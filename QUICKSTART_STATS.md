# Быстрый старт: Статистика прослушивания

## Что было добавлено?

Система автоматического отслеживания времени прослушивания аудиокниг с детализацией по дням, месяцам и годам.

## Как это работает?

**Автоматически!** Просто запустите приложение и слушайте книги как обычно. Система будет:
- ✅ Отслеживать реальное время прослушивания
- ✅ Записывать данные каждые 10 секунд
- ✅ Сохранять статистику по дням для каждой книги
- ✅ Автоматически агрегировать данные по месяцам и годам

## Просмотр статистики

### Вариант 1: Через командную строку

```bash
# Общая сводка
python view_stats.py --summary

# Статистика за последние 7 дней
python view_stats.py --daily

# Топ-10 книг за всё время
python view_stats.py --top

# Месячная статистика
python view_stats.py --monthly

# Годовая статистика
python view_stats.py --yearly
```

### Вариант 2: Программно (Python)

```python
from database import DatabaseManager
from pathlib import Path

db = DatabaseManager(Path("data/audiobooks.db"))

# Статистика за сегодня
from datetime import datetime
today = datetime.now().strftime('%Y-%m-%d')
stats = db.get_daily_stats(start_date=today, end_date=today)

for stat in stats:
    hours = stat['total_seconds'] / 3600
    print(f"{stat['audiobook_name']}: {hours:.2f} часов")
```

## Технические детали

### Что отслеживается?
- **Реальное время** - фактическое время прослушивания (не зависит от скорости воспроизведения)
- **Сессии** - каждый период непрерывного прослушивания
- **Дата и время** - когда и сколько вы слушали

### Где хранятся данные?
- База данных: `data/audiobooks.db`
- Таблицы: `listening_sessions`, `daily_listening_stats`

### Производительность
- Обновление каждые 100ms (в памяти)
- Запись в БД каждые 10 секунд
- Минимальное влияние на производительность

## Важно знать

1. **Новая система** - Существующие данные в поле `listened_duration` не конвертируются. Статистика начинается с момента обновления.

2. **Реальное время** - Если слушаете на скорости 2x, 1 час контента = 30 минут реального времени.

3. **Автосохранение** - Данные сохраняются автоматически. При аварийном завершении может быть потеряно до 10 секунд.

4. **Миграция БД** - При первом запуске автоматически создаются новые таблицы.
